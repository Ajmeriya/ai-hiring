# Question Service

FastAPI microservice for aptitude question generation, question banking, and per-application assignment.

## What it does

- Stores aptitude questions in MySQL per job.
- Generates new questions with Gemini through LangChain when the bank is too small.
- Falls back to local aptitude templates when Gemini is unavailable or fails.
- Assigns a stable set of questions to each application.
- Logs the source of questions so you can see whether they came from `gemini`, `fallback`, or the database.

## Runtime behavior

- Questions are stored in `aptitude_question_bank`.
- Per-application assignments are stored in `aptitude_question_assignments`.
- Each bank row stores `source` and `correct_answer_index`.
- If the bank still contains old generic questions, it is refreshed automatically on the next assignment.

## Environment

Create a `.env` file with:

```env
API_HOST=0.0.0.0
API_PORT=8086
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=
MYSQL_DATABASE=ai_hiring_questions
GEMINI_API_KEY=your_key_here
GEMINI_MODEL=gemini-1.5-flash
POOL_MULTIPLIER=5
```

## Run

```powershell
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8086 --reload
```

## API

- `GET /api/aptitude/health`
- `POST /api/aptitude/questions/assign`

## Request shape

The assign endpoint expects:

- `applicationId`
- `jobId`
- `count`
- `jobTitle`
- `jobDescription`
- `requiredSkills`
- `topics`
- `difficulty`
- `aptitudeType`
- `aptitudeTime`
- `recruiterTopicsRaw`

## Question generation sources

- `gemini`: Generated by Gemini and parsed as JSON.
- `fallback`: Local aptitude generator used when Gemini is unavailable or errors.
- `db-assignment`: Questions already assigned to the application.
- `db-random`: Questions selected from the database bank for a new assignment.

## Troubleshooting

- If the assign endpoint returns fallback questions, check the logs for `Question source=fallback`.
- If `application-service` returns 404 on aptitude start, make sure `job-service` is running and the job ID exists.
- If Gemini fails with `INVALID_PROMPT_INPUT`, check the prompt variables passed from `application-service`.

## Database tables

- `aptitude_question_bank`
- `aptitude_question_assignments`

## Notes

- The question bank keeps a reusable pool per job.
- The service stores the correct answer index with every question so assessments can be scored later.
- Existing application assignments are reused if they already exist.
